# 客戶測試帳號自助申請指南

## 🎯 概述

現在客戶可以透過註冊流程自行申請測試帳號，系統會自動保存帳密資訊供測試使用。

## 📋 申請流程

### 第一步：訪問註冊頁面
1. 打開網站首頁
2. 點擊「申請新帳號」按鈕
3. 進入自助註冊表單

### 第二步：填寫註冊資訊
1. **基本資訊**：
   - 姓名：輸入您的真實姓名
   - Email：使用有效的 Email 地址（需要接收驗證信）
   - 密碼：至少 6 個字元（請設定易記的測試密碼）
   - 確認密碼：重複輸入密碼

2. **角色選擇**：
   - **執行單位操作人員**：一般測試用戶（推薦）
   - **輔導老師**：文化部輔導人員
   - **系統管理員**：最高權限用戶

3. **單位資訊**（操作人員需填寫）：
   - 單位代碼：例如 `UNIT001`
   - 單位名稱：例如 `測試執行單位`

4. **申請理由**（選填）：
   - 簡述申請目的，例如：「系統功能測試」

### 第三步：獲得帳密資訊
提交註冊後，系統將顯示：

```
🔐 請保存您的帳號資訊

帳號 (Email): your-email@example.com
密碼: your-password-123
姓名: 張測試
角色: 執行單位操作人員

⚠️ 請截圖或手動記錄上述帳密資訊，以便日後登入測試
```

### 第四步：Email 驗證
1. 檢查您的信箱（包含垃圾郵件夾）
2. 點擊驗證連結完成帳號啟用
3. 返回登入頁面使用剛才記錄的帳密登入

## 🔧 管理員功能

管理員可以透過「權限管理」>「帳號註冊記錄」查看所有申請的測試帳號：

- **查看所有註冊記錄**：包含帳號、密碼、註冊時間等
- **搜尋功能**：按姓名、Email 或單位搜尋
- **密碼顯示/隱藏**：保護敏感資訊
- **複製功能**：一鍵複製帳號或密碼
- **匯出 CSV**：批量導出所有帳號資料
- **刪除記錄**：清理不需要的測試帳號

## 📊 帳號統計

系統會自動統計：
- 總註冊數量
- 各角色數量（管理員/輔導老師/操作人員）
- 註冊時間分布

## ⚠️ 注意事項

1. **密碼安全**：
   - 測試環境請使用簡單易記的密碼
   - 正式環境需使用強密碼

2. **Email 驗證**：
   - 必須完成 Email 驗證才能登入
   - 驗證信有效期限為 24 小時

3. **帳號管理**：
   - 每個 Email 只能註冊一次
   - 管理員可以查看並管理所有測試帳號

4. **數據保存**：
   - 帳密資訊保存在瀏覽器 localStorage
   - 清除瀏覽器資料會導致記錄遺失
   - 建議定期匯出備份

## 🚀 快速測試帳號範例

以下為建議的測試帳號格式：

```
管理員測試帳號：
Email: admin-test@example.com
密碼: admin123
角色: 系統管理員

輔導老師測試帳號：
Email: coach-test@example.com  
密碼: coach123
角色: 輔導老師

操作人員測試帳號：
Email: operator-test@example.com
密碼: operator123
角色: 執行單位操作人員
單位: 測試執行單位 (TEST001)
```

## 💡 建議

1. **批量申請**：可以一次申請多個不同角色的測試帳號
2. **命名規範**：使用有意義的 Email 和密碼便於記憶
3. **定期備份**：匯出 CSV 檔案保存帳號清單
4. **測試完成後**：及時清理不需要的測試帳號
