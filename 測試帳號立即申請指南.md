# 🚀 立即可用的測試帳號創建指南

## ⚠️ 重要說明

**目前系統中沒有任何預設帳號可以直接登入！**

您有兩個選擇來獲得測試帳號：

## 方案一：自助註冊申請（推薦）

### 🎯 客戶自行申請測試帳號

1. **訪問註冊頁面**
   - 打開系統首頁
   - 點擊「申請新帳號」或「註冊」按鈕

2. **填寫註冊表單**
   ```
   姓名: 測試管理員
   Email: your-email@example.com  
   密碼: test123456
   角色: 系統管理員
   ```

3. **獲得帳密資訊**
   - 註冊成功後系統會顯示：
   ```
   🔐 請保存您的帳號資訊
   帳號: your-email@example.com
   密碼: test123456
   角色: 系統管理員
   ```

4. **完成驗證並登入**
   - 檢查信箱點擊驗證連結
   - 使用剛才記錄的帳密登入

### 📋 建議申請的測試帳號

```bash

角色: 執行單位操作人員
單位代碼: TEST001
單位名稱: 測試執行單位
```

## 方案二：管理員手動創建（需要 Supabase 權限）

### 🔧 適用對象
- 有 Supabase 管理權限的技術人員
- 需要大批量創建測試帳號的情況

### 📝 操作步驟

#### 1. 在 Supabase Dashboard 創建 Auth 用戶
```

點擊: "Add user" 按鈕

創建以下用戶:

│ Auto Confirm User: ✅ (重要!)            │

│ Password: coach123                      │
```

## 🎯 推薦流程 (最簡單)

### ✅ Step 1: 立即申請一個管理員帳號
```
1. 訪問系統首頁
2. 點擊「申請新帳號」  
3. 填寫表單:
   - 姓名: 測試管理員
   - Email: admin-test@gmail.com (使用您的真實郵箱)
   - 密碼: admin123456
   - 角色: 系統管理員
4. 記錄顯示的帳密資訊
5. 檢查郵箱點擊驗證連結
6. 使用記錄的帳密登入
```

### ✅ Step 2: 使用管理員帳號創建更多測試帳號
```
1. 登入後進入「權限管理」
2. 點擊「新增用戶」
3. 創建各種角色的測試帳號
4. 或在「帳號註冊記錄」中查看所有申請的帳號
```

## 📱 目前可用功能

✅ **自助註冊申請** - 客戶可自行申請任何角色的帳號  
✅ **帳密明文顯示** - 註冊成功後清楚顯示帳號密碼  
✅ **管理員帳號管理** - 查看所有註冊的測試帳號記錄  
✅ **匯出帳號清單** - 批量導出 CSV 格式的帳號資料  
✅ **Email 驗證流程** - 完整的帳號啟用機制
