# 🧪 人工驗收測試步驟

## 📋 測試環境
- **正式環境**: https://mocwork.atipd.tw
- **管理員帳號**: mag@atipd.tw / admin123
- **需要建立**: 操作人員和輔導老師測試帳號

---

## 🔐 **測試 1: 管理員登入驗收**

### 步驟：
1. 前往 https://mocwork.atipd.tw
2. 使用 `mag@atipd.tw` / `admin123` 登入
3. 點擊左側選單「**新案提案申請**」

### 預期結果：
✅ **管理員區域（藍色背景）**：
- [ ] 所有欄位可編輯
- [ ] project_code 欄位顯示且為必填
- [ ] 輸入 project_code 時有載入動畫
- [ ] 重複編號顯示「計畫編號已存在」錯誤
- [ ] 顯示權限標籤「可編輯」

✅ **操作人員區域（綠色背景）**：
- [ ] 管理員可以編輯所有欄位
- [ ] 顯示權限標籤「管理員全部可編輯」

✅ **批次填入期程工具**：
- [ ] 勾選「🚀 批次填入期程工具」後展開
- [ ] 方式1：輸入開始日期 + 月數，點擊「🎯 計算並套用」
- [ ] 方式2：輸入結束日期，點擊「📦 套用日期區間」  
- [ ] 確認日期自動填入上方的「計畫開始日」和「計畫結束日」

✅ **儲存測試**：
- [ ] 填寫必填欄位（project_code, 計畫名稱, 執行單位）
- [ ] 點擊儲存按鈕
- [ ] 確認成功儲存並返回計畫列表
- [ ] 在 Supabase 中確認資料已寫入 projects 表

1. 使用新建的操作人員帳號登入
2. 點擊左側選單「**新案提案申請**」

- [ ] 所有欄位顯示為 disabled（灰色背景）
- [ ] 顯示權限標籤「等待管理員開案完成」+ 🔒 圖示
- [ ] 無法編輯任何管理員欄位
- [ ] 看不到批次填入期程工具

✅ **操作人員區域（綠色背景）**：
- [ ] 所有欄位可正常編輯
- [ ] 顯示權限標籤「操作人員可編輯」
- [ ] 可以儲存操作人員負責的欄位

✅ **功能測試**：
- [ ] 填寫計畫描述、預算等欄位

## ⚠️ **常見問題排除**

### 如果 project_code 唯一性檢查不工作：
1. 確認 Supabase SQL 遷移已執行
2. 檢查瀏覽器開發者工具的網路請求
3. 確認 VITE_SUPABASE_URL 和 VITE_SUPABASE_ANON_KEY 環境變數

### 如果權限控制不正確：
1. 確認帳號角色設定正確（profiles 表 role 欄位）
2. 檢查瀏覽器 localStorage 中的使用者資訊
3. 嘗試清除瀏覽器快取並重新登入

### 如果批次日期功能不工作：
1. 確認以管理員身份登入
2. 檢查是否勾選了「批次填入期程工具」
3. 確認開始日期和月數/結束日期都有填寫
