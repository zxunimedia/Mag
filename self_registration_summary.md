## 🚀 系統功能

### 1. 線上自助申請
- ✅ **直接註冊**：用戶可以直接在登入頁面點擊「申請新帳號」
- ✅ **無需管理員**：整個申請過程無需管理員手動操作
- ✅ **即時處理**：系統自動處理註冊請求和確認信發送

### 2. 三種角色支援
- 🎓 **輔導老師 (COACH)**
  - 無需填寫單位資訊
  - 自動歸屬於文化部 (MOC)
  - 顯示角色說明提示
  
- 👥 **操作人員 (OPERATOR)**
  - 必須填寫單位代碼和名稱
  - 代表執行單位的操作人員
  
- 🛡️ **管理員 (ADMIN)**
  - 單位資訊為選填
  - 可能隸屬於不同管理單位

### 3. 智能表單設計
- 🔄 **動態表單**：根據選擇的角色顯示相應欄位
- ⚠️ **即時驗證**：表單填寫時即時檢查和提示
- 📝 **友善介面**：清晰的欄位說明和填寫指引
- 💡 **角色提示**：為每個角色提供專門的說明

## 📊 註冊流程

### 用戶端流程
1. **訪問登入頁** → 點擊「申請新帳號」
2. **填寫申請表** → 選擇角色、輸入基本資訊
3. **提交申請** → 系統自動處理
4. **收到確認信** → 點擊連結啟用帳號
5. **等待審核** → 帳號進入待審核狀態
6. **審核通過** → 可正常登入使用

### 系統端處理
1. **接收申請** → 驗證資料完整性
2. **建立帳號** → Supabase Auth 用戶
3. **建立檔案** → profiles 表記錄
4. **發送確認信** → 自動 Email 確認
5. **設定狀態** → pending_approval
6. **通知管理員** → 待審核列表

## 🔧 技術實現

### 核心組件
- `SelfRegistration.tsx` - 自助註冊表單組件
- `Login.tsx` - 整合註冊入口的登入頁面
- Supabase Auth - 用戶認證和確認信
- profiles 表 - 用戶檔案和審核狀態

### 安全機制
- 🔒 **雙重驗證**：Email 確認 + 管理員審核
- ⚡ **速率限制**：防止惡意註冊攻擊
- 📧 **Email 驗證**：確保真實 Email 地址
- 👥 **審核機制**：管理員控制系統存取權限

### 資料結構
```sql
-- profiles 表新增欄位
status: 'pending_approval' | 'approved' | 'rejected'
registration_reason: TEXT (申請理由)
created_at: TIMESTAMP (註冊時間)
```
