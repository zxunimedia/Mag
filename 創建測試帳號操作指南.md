# 🎯 為客戶創建測試帳號 - 操作指南

```
# 1. 管理員帳號 (最高權限)
Email: admin-test@mocwork.com
密碼: admin123456
角色: 系統管理員

# 2. 輔導老師帳號
Email: coach-test@mocwork.com  
密碼: coach123456
角色: 輔導老師

# 3. 操作人員帳號
Email: operator-test@mocwork.com
密碼: operator123456
角色: 執行單位操作人員
單位: 測試執行單位 (TEST001)
```

## 🚀 創建方式選擇

### 方式一：使用內建的測試帳號創建器 ⭐ (推薦)

1. **訪問創建頁面**
   ```
   直接在瀏覽器中訪問：
   https://your-domain.com/create-test-accounts
   
   或
   
   https://mocwork.atipd.tw/create-test-accounts
   ```

2. **一鍵創建**
   - 頁面會顯示要創建的三個測試帳號
   - 點擊「立即創建測試帳號」按鈕
   - 等待創建完成（約30秒）

3. **獲得帳號資訊**
   - 系統會顯示創建成功的帳號清單
   - 包含完整的帳號密碼資訊
   - 支援一鍵複製功能

### 方式二：手動註冊（如果自動創建失敗）

如果方式一不可用，可以手動註冊這三個帳號：

1. **訪問首頁註冊**
   - 點擊「申請新帳號」
   - 依序填寫上述三個帳號的資訊
   - 每個帳號註冊成功後都會顯示帳密資訊

2. **註冊順序建議**
   ```
   第一個: 管理員帳號 (admin-test@mocwork.com)
   第二個: 輔導老師帳號 (coach-test@mocwork.com)
   第三個: 操作人員帳號 (operator-test@mocwork.com)
   ```

## ⚠️ 重要提醒

### Email 驗證問題
- 這些測試帳號使用的是 `@mocwork.com` 域名
- **客戶無法收到驗證信**（因為不是真實信箱）
- 需要在 Supabase Dashboard 手動確認用戶

### 手動確認用戶步驟：
1. 登入 Supabase Dashboard
2. 進入 Authentication > Users
3. 找到剛創建的三個用戶
4. 點擊每個用戶，勾選 "Email Confirmed"
5. 保存設定

### 或者修改Email使用真實信箱：
如果希望客戶能收到驗證信，建議修改創建的帳號：
```
admin-test@gmail.com (使用您的真實Gmail)
coach-test@gmail.com (使用您的真實Gmail)  
operator-test@gmail.com (使用您的真實Gmail)
```

## 🎯 創建完成後的驗證

### 1. 確認帳號存在
在 Supabase Dashboard > Authentication > Users 中應該看到：
```
✅ admin-test@mocwork.com
✅ coach-test@mocwork.com
✅ operator-test@mocwork.com
```

### 2. 確認 Profiles 記錄
在 Supabase > Table Editor > profiles 中應該看到對應的記錄

### 3. 測試登入
每個帳號都能正常登入且角色正確

## 🔧 故障排除

### 如果創建失敗：
1. 檢查 Supabase 環境變數配置
2. 確認網路連線正常
3. 查看瀏覽器開發者工具中的錯誤訊息

### 如果登入失敗：
1. 確認帳號已在 Supabase 中確認 Email
2. 檢查 RLS 政策是否正確設定
3. 確認 profiles 表中有對應記錄

### 緊急備用方案：
如果所有自動化方式都失敗，可以使用 SQL 手動創建：
- 執行 `create_initial_test_accounts.sql` 腳本
- 或者執行 `create_test_accounts_manual.sql` 腳本
